# Project User Stories

This document outlines the nine core user stories for the Mobile App Development Capstone Project, covering authentication, core features, data persistence, and notifications.

## Authentication
1. **User Registration:** As a new user, I want to create an account by providing my username, email, and password so that I can access the app's personalized features.
2. **User Login:** As a returning user, I want to log in using my email and password so that I can securely access my saved data and preferences.
3. **Input Validation:** As a user, I want to see clear error messages if I enter an invalid email format or leave fields blank, so I can correct my mistakes before submitting.

## Core Functionality
4. **Browse Items (Home):** As a user, I want to view a list of items or categories on the home screen so that I can quickly browse through the available content.
5. **Item Details:** As a user, I want to click on a specific item and be navigated to a detail screen so that I can see more in-depth information about that item.
6. **Navigation:** As a user, I want a clear navigation icon or button on the main screen so that I can easily move between the home and detail views.

## Data & Integration
7. **External API Integration:** As a user, I want the app to display real-time data fetched from an external source so that the information is always up-to-date.
8. **Local Data Persistence:** As a user, I want my preferences or "favorite" items to be saved locally on my device so that they are still available even after I close and reopen the app.

## Settings & Notifications
9. **In-App Notifications:** As a user, I want to receive a test notification or alert when I interact with a specific setting so that I can confirm the app is communicating with me correctly.
